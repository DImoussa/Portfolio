Doctrine DB Mapper Bundle

Bundle Symfony open-source publié sur Packagist

Symfony Bundle Doctrine ORM CLI PHP 8.1+ Packagist

Présentation

Doctrine DB Mapper Bundle (v1.4.1) est un bundle Symfony open-source que j'ai conçu et publié sur Packagist. Il permet de générer automatiquement des entités Doctrine à partir d'une base de données MySQL existante, avec un support complet des relations (OneToMany, ManyToOne, ManyToMany, OneToOne).

Ce bundle est né directement de mon projet GestTravaux Pro : la base de données MySQL était déjà en place, et générer manuellement toutes les entités Doctrine aurait coûté un temps précieux à l'équipe. J'ai développé ce bundle pour automatiser entièrement ce processus — une fois le gain de temps prouvé sur notre projet, je l'ai publié sur Packagist pour le partager à toute la communauté Symfony.

Installable en une seule commande : composer require dimoussa/doctrine-db-mapper-bundle

Fonctionnalités Principales

Génération Automatique

Analyse le schéma MySQL et génère les entités Doctrine avec les types PHP corrects, les annotations ORM et les repositories associés

Détection des Relations

Analyse automatique des clés étrangères pour générer les relations bidirectionnelles : OneToMany, ManyToOne, ManyToMany, OneToOne

Mode Interactif

Interface CLI interactive pour modifier le schéma : ajouter des colonnes, gérer les relations, prévisualiser le SQL avant exécution

Prévisualisation SQL

Système de plan de changements avec prévisualisation SQL complète avant toute modification réelle de la base de données

Getters / Setters / Collections

Génération automatique des getters, setters et méthodes add/remove pour les collections Doctrine

Package Packagist

Distribué en tant que package Composer sur Packagist, installable et utilisable dans n'importe quel projet Symfony

Commandes Disponibles

dbmapper:generate-entities

Génère automatiquement toutes les entités Doctrine depuis la base de données :

php bin/console dbmapper:generate-entities src/Entity
  • Analyse complète du schéma (tables, colonnes, types, contraintes)
  • Détection automatique des clés primaires et étrangères
  • Détection intelligente de l'AUTO_INCREMENT pour les PK
  • Détection des tables de jonction ManyToMany et ordre correct des PK composites
  • Génération des relations bidirectionnelles entre entités
  • Création des repositories avec méthodes de base
  • Synchronisation automatique de la base après génération
  • Option --table=users pour cibler une table spécifique
  • Option --schema-preview pour prévisualiser le SQL avant génération

dbmapper:modify-entities

Mode interactif pour modifier le schéma de la base de données :

  • Lister et explorer les tables et leurs colonnes
  • Ajouter de nouvelles colonnes avec type Doctrine et nullable
  • Créer des relations (ManyToOne, OneToMany, ManyToMany, OneToOne)
  • Plan de changements avec récapitulatif avant exécution
  • Prévisualisation du SQL généré avant application
  • Application sécurisée avec confirmation

Services & Responsabilités

  • SchemaExtractor — Interroge information_schema pour récupérer tables, colonnes, clés primaires et étrangères
  • RelationshipAnalyzer — Détecte les relations entre tables (FK, tables de jonction ManyToMany) et génère les métadonnées Doctrine
  • EntityGenerator — Produit le code PHP des entités avec attributs ORM, types, getters/setters et repositories
  • SchemaExplorer — Fournit une API de lecture interactive du schéma pour la commande modify
  • SchemaChangePlanner — Accumule les changements demandés (colonnes, relations) avant exécution
  • SchemaModifier — Traduit le plan de changements en SQL et l'exécute via Doctrine DBAL
  • DoctrineTypeRegistry — Mappe les types Doctrine vers les types SQL natifs
